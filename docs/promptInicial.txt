Este projeto irá criar uma aplicação desktop nativa para windows que automatiza a extração e catalogação de metadados etnobotânicos a partir de artigos científicos sobre o uso tradicional de plantas por comunidades indígenas e tradicionais. Com esta programa, o pesquisador em etnobotânica irá fazer upload de um artigo científico em PDF e o programa irá, com o auxílio da IA, extrair dados e metadados sobre o artigo em si, e sobre o uso de plantas por comunidades tradicionais. A extração se dará com auxílio da IA, usando o OLLAMA, que estará instalado na máquina do usuário. Os dados serão salvos em um arquivo JSON local, com a seguinte estrutura: @docs\estrutura.json. Este arquivo será usado para alimentar uma base de dados MongoDB, que estará rodando na plataforma ATLAs, ou em um servidor local, que será acessado via URI fornecida na interface de configuração. O programa terá uma interface com uma área para upload do arquivo PDF e informes sobre o processamento da extração dos dados. Terá também uma área de configuração, com espaço para o "prompt" de configuração do modelo de AI, rodando no OLLAMA. NEsta interface de configuração o usuáario irá informar a URI de conexão com o MongoDB. Estes dados ficarão salvos em um arquivo de configuração. HAverá também uma área na interface onde o usuário poderá ver a ficha de todas as refrências já processadas e ele poderá editá-las, deletá-las e mesmo criar um novo registro (interface CRUD). Nesta interface o usuário poderá marcar as fichas para envio ao banco de dados MongoDB. Neste processo, as fichas selecionadas serão enviadas ao MongoDB externo e, se bem sucedido o envio, deletadas do arquivo de dados em JSON local. Logo após o envio e processamento dos dados e metadados do artigo, o usuário deve poder editar todos os dados, e mesmo adicionar novos atributos à ficha do artigo. O usuário deve ser informado que deve fazer o Upload de seu arquivo de dados local para o banco de dados MongoDB remoto, para garantir a segurança dos dados e um bom desempenho do sistema. Considerar um numero máximo de registros para o arquivo local, paraq questões de desempenho, onde não será mais possível enviar novos PDFs para processamento. O programa deve ter um instalador profissional para windows. Os dados mínimos obrigatórios que precisam ser extraídos pelo programa são título, autores, ano de publicação e resumo. Os autores devem ser armazenados no formato APA, o título normalizado em sua "case" e o resumo sempre em português brasileiro. Os dados desejados são publicação (onde foi publicado o artigo), espécies de plantas utilizadas, uso destas plantas, comunidades estudadas, localização, estado, município, bioma e metodologia do estudo. Veja a @docs\estrutura.json. Use a imagem @docs\etnopapers.png para ilustrar o README.md e o programa. O programa deve ter ainda no menu a opção "sobre" com dados do autor - Eduardo Dalcin, Instituto de Pesquisas JaArdim Botânico do Rio de Janeiro, edalcin@jbrj.gov.br. O programa deve informar as conexões bem sucedidas ou não, com o OLLAMA, previamente instalado pelo usuário; e com o MongoDB, fornecido via URI. Destes, o OLLAMA é obrigatório e fundamental para o funcionamento do programa e habilitação de upload do PDF. Nenhum PDF é armazenado localmente. Todos os PDFs enviados serão descartados após o processamento. O programa deve informar ao usuário que o uso de uma placa de vídeo apropriada (GPU) deve ser considerada para melhorar o desempenho da extração de dados pela AI. Crie um README.md para usuários pesquisadores em etnobotânica, todo em português brasileiro. A interface do programa deve ser simples, limpa e moderna.
